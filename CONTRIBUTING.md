# Contributing to SETHX Client

SETHX Client is a community-maintained open-source project. Contributions are welcome.

## Ways to contribute

- Improve documentation and translations
- Report bugs with clear reproduction steps
- Improve UI and accessibility
- Add tests
- Review contract integrations
- Improve transaction review flows
- Help maintain release notes and setup guides

## Development workflow

1. Fork the repository.
2. Create a feature branch.
3. Run `npm install`.
4. Make your changes.
5. Run `npm run build` before opening a pull request.
6. Open a pull request with a clear description.

## Pull request guidelines

- Keep changes focused.
- Explain why the change is needed.
- Include screenshots for UI changes.
- Do not include private keys, seed phrases, RPC secrets, API keys, or personal wallet data.
- Do not change contract addresses or ABIs unless the change is tied to a documented deployment update.

## Maintainers

Maintainers review pull requests, coordinate releases, and help keep the client aligned with the SETHX protocol. Protocol-level governance decisions remain separate from client maintenance.
