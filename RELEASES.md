# Releases

Future SETHX Client releases can include source archives and ready-to-install desktop packages.

Recommended release artifacts:

- source code archive
- Windows installer
- macOS DMG
- Linux AppImage
- checksums file
- release notes

## Release principles

- Releases should be built from public tags.
- Release notes should describe user-facing changes.
- Checksums should be published for downloadable installers.
- Installers should be signed when the project is ready for broad distribution.
- Users should verify downloads before installing.

## Development builds

Development builds are for testing. Users should expect bugs and should verify contract addresses and transaction details carefully.
